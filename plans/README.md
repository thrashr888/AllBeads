# Plans

This directory preserves design plans created during feature development.

## Purpose

Plans document the exploration and decision-making process:

- **Approved plans** - The "why" behind implementations
- **Abandoned plans** - Alternatives considered and reasons for rejection
- **Partial plans** - Exploration that may be revisited

## Naming Convention

`YYYY-MM-DD-feature-name.md`

Example: `2026-01-15-external-plugins.md`

## Relationship to Other Docs

| Doc Type | Purpose                 |
| -------- | ----------------------- |
| Plans    | How decisions were made |
| Beads    | What work was done      |
| Specs    | How the system works    |
| Skills   | How to implement        |
